02.
DEFINITION + IDEATION

Problem Statement + Features Determination

After defining a user insight for our user persona, we finalized the problem statement. This was a vital step in determinig what features would help solve this problem and provide a smooth user experience. We first performed some competitive research to get an idea of what others were using then we utilized an I like, I want, I wish diagram to eventually land on our key features. We developed a feature prioritization matrix from there to focus our UI design efforts on the most impactful and doable features.


User Story + Flow

We wanted to get a better understanding of why and how users would utilize or be directed toward our application, so we started with a simple storyboard that illustrated the users problem and our application as the solution. We then elaborated on this and created a more in depth Customer Journey. This would help us understand the ponits of frustration, happiness and success to ensure our user flows provided a more stremalined UX. 03.
PROTOTYPING + TESTING

Wireframes + User Testing

We focused next on wireframing our application and build a lofi prototype to utilize for user testing. We wanted to make sure we tested in low fidelity first to ensure we captured design and feature changes before adding color and other design elements.


Iterations + Midfi Prototyping

Our usesr test feedback gave us some great insight to needed iterations on our designs. These included: clear tab differentation between sign up and login, adjusting our "favorite" icon to be more distinct, and the addition of a calendar icon to easily set movie watching events with friends or family. After implementing these iterations and running a few more final tests, we focused on completing a mid to hifi version fo FindFlix
